使用Hexo搭建博客

发表于 2013-06-21 | 分类于 爱折腾

生命不息，折腾不止——生命在于折腾。

　　由于不堪忍受wordpress的日益缓慢于臃肿，我终于选择了用静态博客系统来写博客。

　　Hexo 是由Node.js驱动的一款快速、简单且功能强大的博客框架。它和jekyll相比，更快，更轻量。

　　由于Hexo有迁移wordpress上的插件，而wp又有【多说】评论插件，so，我的文章+评论全部都无缝迁移了过来，下面简单做下Mac下搭建Hexo博客的过程。其实最好的教程在官网：http://hexo.io/docs/

##配置环境+从wordpress迁移

  • 安装git：打开终端

ruby -e "$(curl -fsSL https://raw.github.com/mxcl/homebrew/go)"

brew install git

  • 安装Node.js： git clone git://github.com/creationix/nvm.git ~/nvm

　　安装完毕后，在~/.bash_profile加入以下内容并重开一个新的终端视窗。

. ~/nvm/nvm.sh

　　之后即可使用 nvm 来安装 Node.js 了，编译可能需要花些时间，请耐心等候。

  • 安装Hexo：b

npm install -g hexo

  • 生成文件夹

hexo init <folder>即可

##替换Hexo的一些功能

  • 使用多说作为评论系统

　　如果只是单独想使用disqus的话，需要到disqus官网注册，并在_config.yml文件中给disqus_shortname 添加值就可以显示。

　　But，由于我们是在社会主义国家，disqus支持的脸书、推特都很少有人能够登陆，无奈只有操刀把评论系统换为多说。拿到多说的代码，替换/themes/light/layout/_partial/comment.ejs中的代码即可。

　　如果想同时添加多说和disqus,可以参考该链接

  • 使用Jiathis作为社会化分享工具

　　分享的相关代码的位置在themes/light/layout/_partial/article.ejs第35行，将<%-partial('post/share')%>一句删除，并粘贴上Jiathis生成的代码即可。

  • 新浪微博

　　到新浪微博开放平台 生成微博秀代码，将生成的代码保存为weibo.ejs到/themes/light/layout/_widget目录中，并在/themes/light目录下的_config.yml中的widget栏添加- weibo即可。

  • 把Hexo默认的标签改为标签云

　　很简单，在themes/light/_config.yml中，把tag改为tagcloud即可。哈哈
